# 面试话术

## 1. 本地 RAG 链路

我把文档入库拆成上传、清洗、分块、向量化、相似度检索、上下文拼接和答案生成几个步骤。演示版用本地 Hash 向量实现，重点是把 RAG 的工程链路跑通，后续可以把 `embed` 方法替换成真实 Embedding API，把向量存储替换成 Qdrant 或 Milvus。

## 2. 分块与重建索引

文档分块采用段落优先策略，超长段落再用带重叠窗口切分，这样能减少上下文被截断的问题。项目提供重建索引接口，面试时可以解释为什么线上系统需要支持重新切分、重新向量化和灰度重建。

## 3. 鉴权与安全

登录后签发 HMAC token，后端通过 `Authorization: Bearer` 做鉴权。密码不是明文存储，而是加盐 SHA-256。Token 密钥通过环境变量配置，避免把生产密钥写死在代码里。

## 4. 可观测与可维护

新增了知识库仪表盘，能看到文档数、片段数、问答数和最新问题。统一异常返回让前端和测试能稳定处理错误。测试覆盖了注册、文档入库、问答、分块查看、重建索引和删除。

## 5. 可以继续扩展

这个项目可以扩成企业知识库：接入 PDF/Word 解析、对象存储、Qdrant、Redis 缓存、真实大模型 API、问答引用高亮和多租户权限。
