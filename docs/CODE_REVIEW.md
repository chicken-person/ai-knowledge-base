# 代码结构体检与改进记录

## 已发现的问题

- 单文件应用承载 Controller、Service、DTO、鉴权和业务逻辑，适合快速演示，但长期维护应拆成 `controller`、`service`、`repository`、`dto`、`common` 包。
- 原接口异常主要依赖 Spring Boot 默认错误体，前端无法稳定读取业务错误信息。
- Token 默认密钥为固定占位值，容易被误带到生产环境。
- 文档入库缺少删除、重建索引、分块查看和仪表盘统计，RAG 链路不够完整。
- 文档文件名没有做路径清洗，虽然不落盘，也容易在展示和日志中留下脏数据。
- 缺少覆盖核心链路的自动化测试。

## 已完成的修复

- 新增统一异常处理，所有业务错误返回统一 `ApiResponse`。
- `APP_TOKEN_SECRET` 支持环境变量覆盖，H2 控制台也支持 `H2_CONSOLE_ENABLED` 开关。
- 新增文档删除、分块查看、重建索引、知识库仪表盘接口。
- 文档文件名做路径清洗，避免展示 `../xxx` 这类不干净输入。
- Token 解析补充 payload 结构校验。
- 分块策略改为段落优先，超长段落再做带重叠窗口切分。
- 增加常用查询索引：文档列表、分块列表、问答历史。
- 增加集成测试，覆盖注册、入库、问答、分块、重建索引、删除。

## 后续可继续优化

- 拆包重构，替换 `JdbcTemplate` 手写 SQL 为 MyBatis/JPA 均可。
- 接入真实 Embedding 模型和 Qdrant/Milvus。
- 增加文档解析能力，如 PDF、Word、Markdown。
- 增加限流、审计日志和更细粒度的用户权限。
